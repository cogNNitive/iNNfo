Read the Docs About the Project

What is cogNNitive?

A monorepo that ties together the iNNv0 specification ecosystem: the Launcher app, the format-core library, and all specs.

cogNNitive Launcher

Drop a .md file or folder and the Launcher instantly detects whether it's a FILE mode or FOLDER mode FORMAT model, then routes you to the correct editor.

@innv0/format-core

Shared TypeScript library — parser, resolver, and validator used by both the FORMAT app (FILE mode) and iNNfo app (FOLDER mode).

Spec Chain

Four levels: defiNNe (meta) → FORMAT (spec) → Templates (business, procedures, kb) → Models. Each level adds structure over the last.

How the Launcher works

Three simple steps from file to editor.

1

Drop or select

Drag a .md file or an entire folder onto the Launcher, or use the file/folder picker buttons.

2

Detect mode

The Launcher parses the FORMAT frontmatter and identifies FILE, FOLDER, or BOTH mode.

3

Launch editor

Click to open in the matching editor — file-format for single documents, folder-format for node-based models.

Ecosystem Architecture

The four-level spec chain that powers every FORMAT model.

Level 0

defiNNe

The meta-specification. Defines structure, versioning, and RFC 2119 conventions for the entire ecosystem.

Level 1

FORMAT

The central specification. Two modes: FILE (single document) and FOLDER (node-as-folder with assets). Unified relationship system.

Level 2

Templates

Domain-specific templates: business, procedures, kb (knowledge base). Each declares concepts, markers, and matrices.

Level 3

Models

Concrete data instances like Ghostbusters (business) or TeamKB (knowledge base). Lightweight — just data and parent pointer.
