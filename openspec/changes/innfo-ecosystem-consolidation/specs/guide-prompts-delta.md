# Delta: guide-prompts — innfo: Prefix in Guide Prompts

## Change Type

**Modified capability** (cross-cutting): The AI Guide prompt templates in `guide.ts` (`extractPrompt()`) and inline examples in `procedure_NN.md` are updated to use the `innfo:` prefix. These are not standalone panels but make up the Guide tab content rendered by `AIGuidePanel.vue`.

## Old Behavior

### guide.ts — extractPrompt() return values (3 cases)

```typescript
// Edit model case:
return 'Load the innv0-innfo skill — I need to edit a model'

// Import case:
return 'Load the innv0-trannsform skill — I need to import documents from traNNsform/input/ and convert them to iNNfo models'

// Export case:
return 'Load the innv0-innfo skill — I need to generate an HTML visualizer following traNNsform/AGENT.md'
```

### procedure_NN.md — Inline instruction examples (3 lines)

```
Line 48: *"Load the innv0-innfo skill and check that innfo-mcp is configured"*
Line 57: *"Load the innv0-innfo skill — I need to edit the business model and add a new concept"*
Line 75: *"Load the innv0-innfo skill — I need to generate an HTML visualizer following traNNsform/AGENT.md"*
```

## New Behavior

### guide.ts — extractPrompt() return values

All three return values MUST be wrapped with `innfoPrompt()` from `prompt.ts`:

| Case | Old return value | New return value (wrapped) |
|------|------------------|---------------------------|
| edit model | `'Load the innv0-innfo skill — I need to edit a model'` | `'innfo: Load the innv0-innfo skill — I need to edit a model'` |
| import | `'Load the innv0-trannsform skill — I need to import documents...'` | `'innfo: Load the innv0-trannsform skill — I need to import documents...'` |
| export | `'Load the innv0-innfo skill — I need to generate an HTML visualizer following traNNsform/AGENT.md'` | `'innfo: Load the innv0-innfo skill — I need to generate an HTML visualizer following traNNsform/AGENT.md'` |

### procedure_NN.md — Inline instruction examples

Three lines in `procedure_NN.md` MUST be updated to use the `innfo:` prefix:

| Line | Old text | New text |
|------|----------|----------|
| 48 | `*"Load the innv0-innfo skill and check that innfo-mcp is configured"*` | `*"innfo: Load the innv0-innfo skill and check that innfo-mcp is configured"*` |
| 57 | `*"Load the innv0-innfo skill — I need to edit the business model and add a new concept"*` | `*"innfo: Load the innv0-innfo skill — I need to edit the business model and add a new concept"*` |
| 75 | `*"Load the innv0-innfo skill — I need to generate an HTML visualizer following traNNsform/AGENT.md"*` | `*"innfo: Load the innv0-innfo skill — I need to generate an HTML visualizer following traNNsform/AGENT.md"*` |

Note: Line 75 still references `traNNsform/AGENT.md` in the instruction text. This is an example that demonstrates what the user types — the actual prompt generated by the ExportPanel will use `workflows/export.workflow.md`. The example in the guide instructs the user, so it stays as AGENT.md for now (keeping consistency with the guide's educational purpose). If changed later, both the procedure_NN.md text AND the ExportPanel prompt must agree.

## Requirements

### R-GD-D01: guide.ts Uses innfoPrompt()

The `extractPrompt()` function in `guide.ts` MUST wrap all three return values with `innfoPrompt()`. This is done by modifying the `return` statements — no structural changes to the function.

#### Scenario: All three guide prompts have innfo: prefix
- GIVEN a model is open and the Guide tab is active
- WHEN the user expands an edit/import/export step
- THEN the copyable prompt starts with `innfo:`
- AND clicking Copy copies the `innfo:` prefixed text

### R-GD-D02: procedure_NN.md updated inline

The `procedure_NN.md` source file at `src/ai-guide/procedure_NN.md` MUST have its three example instruction lines updated with the `innfo:` prefix. This is a direct string replacement — no structural changes to the markdown.

#### Scenario: Guide renders innfo:-prefixed examples
- GIVEN the Guide tab renders procedure_NN.md content
- WHEN the user reads the step descriptions
- THEN the example instruction texts show `"innfo: Load..."` instead of `"Load..."`

## Acceptance Criteria

- [ ] `guide.ts` `extractPrompt()` return values all use `innfoPrompt()`
- [ ] Line 48 of `procedure_NN.md` starts with `innfo:`
- [ ] Line 57 of `procedure_NN.md` starts with `innfo:`
- [ ] Line 75 of `procedure_NN.md` starts with `innfo:`
- [ ] All three prompt strings in `guide.ts` produce `innfo: ...` when copied from the Guide tab
- [ ] The `innfo:` prefix is applied at the source (not at render time)
